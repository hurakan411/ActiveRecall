# MindRecaller：詳細要件定義書（v2.0 - 柔らかいフラットデザイン版）

AIを活用したアクティブリコール（記憶の強制想起）を完全自動化するiOSアプリ。データはユーザーのデバイス内に安全に保存され、新規登録またはライブラリからの選択により、何度でも繰り返し学習可能です。

---

## 1. システム構成（Tech Stack）

| 区分 | 技術 | 備考 |
| :--- | :--- | :--- |
| **Frontend** | Swift (SwiftUI) | iOSアプリ本体。AVFoundation, PhotosUI活用。 |
| **Database** | SwiftData (SQLite) | ユーザーのデバイス内にデータを保存。教材とログの1対多関係を保持。 |
| **Local Storage** | File System | 教材画像などはアプリ内のドキュメントディレクトリに保存。 |
| **Backend** | Python (FastAPI) | Render.comでホスト。Gemini APIへの中継を行うステートレス構成。 |
| **AI (LLM/OCR)** | Gemini 2.0 Flash | OCR、要約、セマンティックマッチング（想起採点）を担当。 |

---

## 2. 画面別要件

### 2.1 【画面1】ホーム画面 (Home)
- **目的**: 学習の動機付けと、学習フローへのエントランス。

### 2.2 【画面2】教材ライブラリ画面 (Material Library)
- **目的**: 登録済み教材の閲覧・管理・編集。

### 2.3 【画面3】インプット選択・登録画面 (Input Selection & Registration)
- **目的**: 学習ソースの決定と、正確な「正解データ」の構築。
- **自動入力エディタ**: Geminiが抽出したテキストが編集可能な状態で表示される。タイトルも自動生成。

### 2.4 【画面4】アクティブリコール実行画面 (Recall Session)
- **目的**: 記憶のアウトプット。
- **想起入力**: テキストまたは音声。タイトルのみがヒントとして表示される。

### 2.5 【画面5】解析結果画面 (Analysis Result)
- **目的**: 採点結果の表示と、記憶の「穴」の可視化。

## 3. データモデル（SwiftData）

### 3.1 `Material` モデル
- `id`: UUID
- `title`: String
- `sourceText`: String
- `createdAt`: Date
- `logs`: `[StudyLog]`

### 3.2 `StudyLog` モデル
- `id`: UUID
- `material`: `Material?`
- `logicScore`: Int
- `termScore`: Int
- `recallText`: String
- `logicFeedback`: String
- `missingKeywords`: `[String]`
- `missingConcepts`: `[String]`
- `createdAt`: Date

---

## 4. AI解析ロジック（Gemini 2.0 Flash / Backend）

### 4.1 インプット解析エンドポイント
- **役割**: 画像/音声からタイトルと詳細テキストを構造化して抽出。

### 4.2 想起採点エンドポイント
- **入力**: `sourceText` + `recallText`
- **出力**: JSON (Scores, Feedback, Gaps)

## 5. UI/UX・デザイン定義

### 5.1 デザインコンセプト
- **コンセプト**: 「Calm Clarity（穏やかな明晰さ）。」
- **視覚的特徴**:
    - **ソフト・フラット**: 装飾を削ぎ落としたフラットデザインをベースにしつつ、色調を中間色（ミューテッドカラー）に統一し、心理的なハードルを下げる。
    - **低コントラスト**: 白と黒の強い対比を避け、オフホワイトやチャコールグレー、淡いニュアンスカラーを使用。
    - **余白の呼吸**: 情報密度を下げ、一画面で処理する情報量を絞ることで、学習の集中力を維持する。

### 5.2 カラースキーム（柔らかい中間色）

| 用途 | カラー名 | Hex/仕様 |
| :--- | :--- | :--- |
| **Primary** | ソフトデニム | `#64748B` (落ち着いた青) |
| **Secondary** | セージグリーン | `#869D7A` (目に優しい緑) |
| **Success** | ミスティグリーン | `#A7C4BC` (達成感を与える淡い緑) |
| **Warning** | テラコッタ / サンド | `#E2A676` (注意を促す柔らかな橙) |
| **Border** | ウォームグレー | `#E5E7EB` (馴染みの良い境界線) |
| **Background** | アイボリー / ボーン | `#F9F8F6` (背景), `#FFFFFF` (面) |

### 5.3 主要UIコンポーネント
- **ソフト・フラットカード**: 背景は白（#FFFFFF）。境界線（1pt）は非常に淡いグレーを使用。角丸は 16pt と少し広めに設定し、角の強さを取る。
- **ボタン**: ベタ塗りではなく、淡い色の「面」に濃い色の「文字」を載せるスタイル。
- **タイポグラフィ**: フォントサイズは大きめに確保。色は純黒ではなく、深いグレー（#334155）を使用して読みやすさを向上。

### 5.4 アニメーション・遷移
- **フェードイン・フェードアウト**: 要素の出現はパッと切り替わるよりも、わずかな時間をかけて柔らかくフェードする挙動。
- **触覚フィードバック**: 強い振動ではなく、トントンという軽い「タップ感」を多用し、心地よさを優先。

### 5.5 アクセシビリティ・UX配慮
- **視覚疲労の軽減**: 長時間の学習でも目が疲れにくい配色設計。
- **形状の補助**: スコアの良し悪しを色だけでなく、アイコン（笑顔のマークやチェックなど）の形状でも伝える。
